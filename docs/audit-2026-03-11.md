## 1. Indexed Visibility/Scope Tags

**Spec requirement** (nostr-memory-spec.md): All memory events MUST include `visibility` and `scope` tags alongside the d-tag for relay-side filtering.

| Event path | Tags emitted | Status |
|------------|-------------|--------|
| `lib.rs` — `Nomen::store()` (main store/publish) | `visibility`, `scope` | ✅ PASS |
| `consolidate.rs` — consolidation publish | `visibility`, `scope` via `extract_visibility_scope()` | ✅ PASS |
| `send.rs` — messaging events (kind 14, 9, 1) | N/A (not memory events) | ✅ N/A |
| `consolidate.rs` — NIP-09 deletion (kind 5) | N/A (deletion events) | ✅ N/A |

**Helper function**: `memory::extract_visibility_scope()` correctly parses v0.2 d-tags, falls back to `("public", "")` for legacy formats. ✅

**WARN**: `lib.rs:store_direct_with_author()` (used by cluster fusion) does NOT publish to relay — it stores locally only. No Nostr tags emitted. This is fine for local-only cluster memories, but if cluster memories are ever synced to relays, they'll need the tags too.
